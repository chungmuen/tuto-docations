Automate your documentation with GitHub Actions

Created 12th December 2019 by Terazus

Introduction:

In this tutorial we are going to explore source code documentation through continuous integration using a set of open source tools and a simple example.
The first part of this tutorial will focus on setting up the project and writing the code and its documentation.
The second part will focus on how to continuously integrate your documentation. It will be language agnostic and help you to set up a pipeline to automate the generation of the documentation.

  • Time: 1h
  • Difficulty: medium
  • Stack: JavaScript
  • Tools: jsdoc (first part), GitHub (second part)
  • Requirements: have a GitHub account; have NPM and Git installed.

During the course of this tutorial, we are going to create a small DNA feature containing a name and a sequence, and represent it on the DOM as a card.
We will then create two buttons: one two trigger the transcription of the sequence into a RNA sequence, the other to trigger its translation into a protein primary sequence.

Setting up the project:

The final step of this tutorial will involve using GitHub Actions, so if not already not, head to github.com and create a new account. You can then create a new project and clone it locally. We will not go through this process as there are really plenty of tutorials out there.
At the root of your project, create:

  • A file named package.json
  • A file named documentation.conf.json
  • A directory named src
  • Inside src create:
    • A file named index.html
    • A file directory named components
